## 1. 재미 요소 (Fun Factors)

### 1.1 핵심 재미 (Core Fun)
| 요소 | 설명 | 강도 |
|------|------|------|
| **연속 레이드** | 보스를 연속 공략하며 난이도와 보상이 상승 | 높음 |
| **턴제 전투** | 스킬 운용과 파티 조합으로 전략적 전투 | 높음 |
| **시즌 이벤트** | 시즌 목표 달성과 한정 보상 획득 | 높음 |
| **성장/수집** | 캐릭터/장비 강화 및 컬렉션 완성 | 중간 |

### 1.2 만족감 요소 (Satisfaction)
- **보스 격파 보상**: 레이드 티어 상승 및 희귀 보상 획득
- **시즌 완주 보상**: 이벤트 패스/시즌 패스 보상 수령
- **파티 성장**: 레이드 난이도 돌파 체감

### 1.3 도전 요소 (Challenge)
- **연속 레이드 난이도 상승**: 티어별 보스 패턴 강화
- **시즌 목표**: 기간 내 목표 달성 요구
- **파티 구성**: 역할/속성 조합 최적화

### 1.4 사회적 요소 (Social)
- **협동 레이드**: 길드/파티 단위 공략
- **리더보드**: 시즌 점수 랭킹 경쟁
- **길드 보상**: 공동 목표 달성 보상

## 2. 게임 루프 (Game Loop)

### 2.1 코어 루프 (10~20분)
```
[레이드 입장] -> [턴제 전투] -> [보스 격파/실패] -> [보상 획득] -> [파티 강화]
```

### 2.2 세션 루프 (일일)
1. 접속 및 일일 보상 수령
2. 시즌/이벤트 미션 확인
3. 레이드 연속 진행
4. 보상 정산 및 파티 강화

### 2.3 메타 루프 (시즌)
```
[시즌 시작] -> [레이드 티어 상승] -> [한정 보상 획득] -> [시즌 종료] -> [리셋/다음 시즌]
```

## 3. BM 모델 (Business Model)

### 3.1 수익화 채널
| 채널 | 비중 | 설명 |
|------|------|------|
| **이벤트 패스** | 35% | 시즌/이벤트 프리미엄 보상 |
| **한정 캐릭터/스킨** | 30% | 이벤트 한정 캐릭터/스킨 |
| **레이드 패키지** | 20% | 레이드 보조 아이템 패키지 |
| **시즌 번들** | 10% | 시즌 테마 아이템 번들 |
| **리워드 광고** | 5% | 보상형 광고 중심 |

### 3.2 과금 포인트
- 이벤트 패스(프리미엄/프리미엄+)
- 레이드 패키지/추가 입장권
- 한정 캐릭터/스킨 직구매 또는 가챠

### 3.3 광고 정책
- 보상형 광고 중심
- 전면 광고는 기본 비활성, 필요 시 비레이드 콘텐츠에 제한적 적용

## 4. 레벨 디자인 (Level Design)

### 4.1 레이드 티어 구조
| 티어 | 난이도 | 주요 보상 |
|------|------|----------|
| T1 | 낮음 | 기본 재화, 강화 재료 |
| T2 | 중간 | 레이드 토큰, 희귀 재료 |
| T3 | 높음 | 한정 조각, 패스 경험치 |
| T4 | 매우 높음 | 한정 장비/스킨 |
| T5 | 최상 | 시즌 최고 보상 |

### 4.2 난이도 상승 규칙
- 티어 상승 시 보스 패턴 추가 및 체력/공격력 증가
- 시즌 진행도에 따라 보상 단계 확장

---

## 5. UI/UX 설계

### 5.1 핵심 화면
- 로비: 시즌 상태, 이벤트 진행도
- 레이드: 파티 편성, 티어 선택
- 결과: 보상 정산 및 다음 단계 안내
- 상점/패스: 패스 보상 및 번들 노출

### 5.2 정보 계층
1. 레이드 티어/진행도, 시즌 포인트
2. 재화(골드/레이드 토큰/보석)
3. 패스 진행도 및 미션

## 6. 프로덕션 체크리스트

- 레이드 전투/보스 패턴 구현
- 티어별 보상 테이블 검증
- 시즌 이벤트/패스 루프 연결
- 보상형 광고 배치 검증
